About sprig

sprig renders HTML on the server and hydrates only the interactive bits as islands.

  • Static component — a folder with just template.html.
  • Island — add a logic.ts; the folder hydrates on the client.
  • Styles — drop a styles.css into the folder (scoped).

Why sprig

  • Server-rendered HTML by default, zero JS for static pages.
  • Islands hydrate only the interactive parts of a page.
  • Angular-flavoured templates compiled to fast Preact.
