# Consume recipes & the verify loop

## The consume recipe

This ships inside the artifact (`consume/prototype.md`) and tells the `prototype` skill how to apply the brand.

### `consume/prototype.md`
`prototype` builds ONE self-contained `.html`, CDN-only, no build. The head block:

```html
<link href="https://cdn.jsdelivr.net/npm/daisyui@5" rel="stylesheet" type="text/css" />
<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>
<script src="https://cdn.jsdelivr.net/npm/lucide@latest/dist/umd/lucide.min.js"></script>
```
Then paste the contents of `theme.cdn.css` into a `<style>`, set `<html data-theme="brand">`, build
with daisyUI semantic classes (see `components.md`), and call `lucide.createIcons()` after render.

> **Critical gotcha — both CDN tags are required.** The daisyUI stylesheet provides *component* and
> *color* classes (`btn`, `card`, `bg-primary`, `text-base-content`). It does **not** provide Tailwind
> *layout utilities* (`flex`, `grid`, `grid-cols-3`, `w-60`, `gap-6`, `p-6`, `hidden`, `md:flex`). Those
> are generated by `@tailwindcss/browser@4`. Omit the Tailwind browser script and the components will
> theme correctly but the **layout collapses** (sidebars stack, grids linearize). This is the #1
> failure mode; it is invisible in the markup and only shows up when you render.

## The verify loop — render it, don't trust it

Markup that "looks right" routinely renders wrong (the Tailwind gotcha above, charts that don't mount,
a dark palette that's illegible). So **always render `showcase.html` and look.**

1. **Serve over HTTP.** The Playwright MCP blocks `file://`. Start a static server rooted at the
   artifact folder:
   ```bash
   python3 -m http.server 8787 --directory spec/ui/design-system &
   ```
2. **Navigate** to `http://localhost:8787/preview/showcase.html`, **resize** to ~1280×900.
3. **Wait** ~2–3s so the Tailwind browser compiler scans the DOM and ApexCharts mounts; confirm with a
   quick `browser_evaluate` (e.g. `!!document.querySelector('#sales svg')`).
4. **Screenshot** (`fullPage`). Then toggle `data-theme` to `brand-dark` (flip the page's theme
   toggle, or `document.documentElement.setAttribute('data-theme','brand-dark')` + re-render charts)
   and screenshot again.
5. **Look** at both. Layout intact? Primary color on the CTA/KPIs? Charts themed? Dark legible? Fix
   `theme.css` (and re-derive) or the markup, then re-render. Stop the server when done.

What "good" looks like: the brand's primary on the main CTA and headline numbers, the display font on
headings, rounded cards, status badges in the right semantic colors, charts in the brand palette, and
dark mode reachable from the same `data-theme` swap with nothing illegible.
